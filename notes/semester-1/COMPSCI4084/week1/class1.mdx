# ProgSD 课程介绍
Programming & System Development (M) - 2025-2026

## 学习成果（Learning Outcomes）

<Tabs items={["编程环境", "开发技能", "系统管理"]}>
  <Tab value="编程环境">
    1. 熟悉现代编程环境与工作流程；掌握命令行与编辑器使用；理解良好文档的重要性。
    2. 能够配置并使用常见开发工具（IDE, 版本控制, 包管理）。
  </Tab>
  <Tab value="开发技能">
    3. 使用 Java 与 Python 编写结构化程序, 能够基于在线资源独立解决问题。
    4. 掌握程序测试与调试, 能编写基本单元测试。
    5. 熟练使用 Git 等版本控制工具并理解协作流程。
    6. 会使用 Jupyter/Colab/Conda 等数据科学工具并能进行基础数据库操作（SQLite）。
  </Tab>
  <Tab value="系统管理">
    7. 理解 Linux 基础操作与文件系统导航；能进行基本系统监控与打印操作。
    8. 能安装和配置常用软件, 并编写/运行简单的 Shell 脚本。
  </Tab>
</Tabs>

## 课程内容安排

| 周次（日期） | 主题（重点） |
|--------------|--------------|
| Week 1 (23–26 Sep 2025) | 课程介绍：Python 基础, 决策结构, OOP, Git, Conda, Google Colab。 |
| Week 2 (30 Sep; 1, 3 Oct 2025) | Python 进阶：循环, 函数, 数据结构。 |
| Week 3 (7, 8, 10 Oct 2025) | 数据处理与可视化：NumPy, Pandas, Matplotlib；创建自定义包。 |

| Week 6 (28, 29, 31 Oct 2025) | 编程评估（Programming Assessment, 监考）。 |
| December (确切日期见课程公告) | 实验考试（Lab Exam, 开卷）。 |

## 评估方式（Assessments）

<Callout type="warn">
重要：所有评估均为必需。
</Callout>

### 评估组成（简表）

| 评估项目 | 权重 | 日期 | 说明 |
|----------|------:|------|------|
| 同行评议参与 (tAPP) | 5% | 持续进行 | 在Lecture上考, 一共5次, 一周一次, 是团队应用练习的同行评审 |
| 评估练习 1 (Assessed Exercise) | 10% | 2025-10-03 | Python 编程（监考）, 20道题, 1小时 |
| 评估练习 2 (Assessed Exercise) | 15% | 2025-10-10 | Python 编程（监考）, 30道题, 1小时 |
| 评估练习 3 (Assessed Exercise) | 15% | 2025-10-24 | Java 或 Unix（监考）, 30道题, 1小时 |
| 编程评估 (Programming Assessment) | 15% | 2025-10-31 | 综合编程能力, 2道题, 1小时 |
| 实验考试 (Lab Exam) | 40% | 2025-12-11 | 开卷考试：指的是在编写代码时可以参考学习资料 |

<Callout type="error">
说明：评估练习合计 40%；实验考试占 40%, 请重点准备。
</Callout>

## 改良团队学习法（Modified Team-Based Learning）

<Tabs items={["课前准备", "课堂活动", "实验辅导"]}>
  <Tab value="课前准备">
    1. 预习指定材料并完成短测验。
    2. 重点阅读概念和示例代码, 准备课堂讨论。
  </Tab>
  <Tab value="课堂活动">
    1. 个人思考：先独立完成问题。
    2. 小组讨论：与团队成员交流与整合方案。
    3. 团队提交：形成团队答案并进行同行评议。
  </Tab>
  <Tab value="实验辅导">
    1. 实验时间通常为 2 小时；实验材料至少提前一周发布。
    2. 团队内互助完成实验任务；课后提供参考解答与示例代码。
  </Tab>
</Tabs>
